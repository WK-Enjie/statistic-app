📋 Collecting, Classifying and Tabulating Data

What is Data?

Data refers to facts, figures, or information collected for analysis. In statistics, we collect data to understand patterns and make decisions.

Types of Data

📊 Quantitative (Numerical)

Data that can be measured and expressed as numbers.

  • Discrete: Countable values (e.g., number of students, goals scored)
  • Continuous: Measurable values (e.g., height, weight, time)

📝 Qualitative (Categorical)

Data that describes qualities or categories.

  • Nominal: Categories with no order (e.g., colours, blood type)
  • Ordinal: Categories with order (e.g., satisfaction rating)

Methods of Data Collection

1 Surveys / Questionnaires: Collecting data through questions (e.g., asking students their favourite subject)
2 Observation: Recording data through watching (e.g., counting cars at a junction)
3 Experiments: Collecting data through controlled tests (e.g., measuring plant growth)
4 Existing Records: Using data already collected (e.g., census data, school records)

Organising Data

Raw data is unorganised. We classify and tabulate data into:

  • Tally charts — using strokes to count
  • Frequency tables — organising data with their frequencies
  • Grouped frequency tables — grouping data into class intervals
📌 Key Terms:
Population: The entire group being studied
Sample: A subset of the population
Frequency: The number of times a value occurs

📑 Frequency Tables

A frequency table shows how often each value (or range of values) appears in a data set.

Shows frequency of each individual value.

Groups data into class intervals. Useful for continuous or large-range data.

Shows running total of frequencies. Essential for finding median, quartiles from grouped data.

How to Create a Frequency Table

1 List all values (or define class intervals for grouped data)
2 Tally — go through data and mark each occurrence
3 Count — total the tally marks for each value/class

Grouped Frequency Tables

Class Interval: e.g., 40–49, 50–59, 60–69
Class Width: Upper boundary − Lower boundary (e.g., 49.5 − 39.5 = 10)
Class boundaries: e.g., for 40–49, boundaries are 39.5 and 49.5
Mid-value: (Lower + Upper) / 2 (e.g., (40+49)/2 = 44.5)

📊 Bar Graphs

Bar graphs use rectangular bars to represent data. The length/height of each bar is proportional to the value it represents.

Key Features

  • Bars are separated by equal gaps
  • Used for discrete/categorical data
  • Can be vertical or horizontal
  • Both axes must be labelled with a title

✅ Advantages

  • Easy to read and compare
  • Good for categorical data
  • Shows differences clearly

❌ Disadvantages

  • Not suitable for continuous data
  • Can be misleading if scale doesn't start at 0
  • Hard to show exact values

🖼️ Pictograms

Pictograms use pictures or symbols to represent data. Each symbol represents a certain quantity.

Key Features

  • A key/legend must be provided showing what each symbol represents
  • Part-symbols (e.g., half) can represent fractional amounts
  • Symbols must be the same size

✅ Advantages

  • Visually attractive and easy to understand
  • Good for comparing simple data
  • Suitable for young audiences

❌ Disadvantages

  • Hard to represent exact values
  • Part-symbols can be confusing
  • Not suitable for large data sets

📈 Line Graphs

Line graphs display data points connected by straight lines. Best for showing trends over time.

Key Features

  • X-axis usually shows time
  • Points are connected to show trends
  • Can show increase, decrease, or constant patterns

✅ Advantages

  • Shows trends and changes over time clearly
  • Easy to interpolate between data points
  • Can compare multiple data sets on same graph

❌ Disadvantages

  • Only suitable for continuous data over time
  • Can be misleading with inappropriate scales
  • Connecting lines may imply continuity where data is discrete

🥧 Pie Charts

Pie charts show data as slices of a circle. Each slice represents a proportion of the whole.

Key Features

  • Total of all slices = 360°
  • Angle of each slice = (value / total) × 360°
  • Good for showing proportions
Angle = (frequency / total frequency) × 360°

✅ Advantages

  • Shows proportions clearly
  • Easy to see which category is largest
  • Visually appealing

❌ Disadvantages

  • Hard to compare similar-sized slices
  • Not useful for many categories
  • Doesn't show actual values without labels

⚫ Dot Diagrams

A dot diagram (dot plot) uses dots above a number line to show the frequency of each data value.

Key Features

  • Each dot = one data value
  • Dots stacked above the number line
  • Easy to see mode, clusters, gaps, and outliers

✅ Advantages

  • Simple to draw and read
  • Shows every data point
  • Easy to identify mode, clusters, gaps

❌ Disadvantages

  • Not suitable for large data sets
  • Becomes cluttered with many values
  • Not ideal for continuous data

📶 Histograms (Equal Class Intervals)

Histograms display grouped continuous data using bars with no gaps between them.

Histogram vs Bar Graph

📊 Bar Graph

  • Gaps between bars
  • For discrete/categorical data
  • Bars can be in any order

📶 Histogram

  • No gaps between bars
  • For continuous data grouped in class intervals
  • Bars must be in order
  • Area of bar ∝ frequency

✅ Advantages

  • Shows distribution shape (symmetric, skewed)
  • Good for large continuous data sets
  • Shows frequency of intervals clearly

❌ Disadvantages

  • Individual values are lost
  • Different class widths change appearance
  • Harder to read exact frequencies

🌿 Stem-and-Leaf Diagrams

A stem-and-leaf diagram splits each data value into a stem (leading digits) and leaf (last digit).

How to Read

For stem = 4 and leaf = 5, the value is 45.

Key: 4 | 5 means 45
Leaves should be in ascending order from the stem outward.

Example

Stem | Leaf
45 8
53 5
60 5 7 8
72 3 7 9
82 5 8
91

✅ Advantages

  • Retains all original data values
  • Shows shape of distribution
  • Easy to find median and mode
  • Good for small to medium data sets

❌ Disadvantages

  • Not suitable for very large data sets
  • Only works for numerical data
  • Can be hard for very spread-out data

📈 Cumulative Frequency Diagrams

A cumulative frequency curve (ogive) shows the running total of frequencies. Used to find median, quartiles, and percentiles.

How to Draw

1 Create a grouped frequency table
2 Calculate cumulative frequency (running total)
3 Plot points at upper class boundaries against cumulative frequency
4 Join the points with a smooth S-shaped curve
📌 Remember: Always start the curve from the lower boundary of the first class (where CF = 0). Plot at upper boundaries.

Reading from the Curve

  • Median: Read at CF = n/2
  • Q1: Read at CF = n/4
  • Q3: Read at CF = 3n/4
  • Percentile Pk: Read at CF = kn/100
  • IQR: Q3 − Q1
Median from curve: read x-value at y = n/2
Q1: read x-value at y = n/4
Q3: read x-value at y = 3n/4
IQR = Q3 − Q1

📦 Box-and-Whisker Plots

A box-and-whisker plot (box plot) summarises data using five key values: minimum, Q1, median, Q3, maximum.

Five-Number Summary

MinimumMin
Q125th%
MedianQ2
Q375th%
MaximumMax

Interpreting the Plot

  • The box contains the middle 50% of data (IQR)
  • The line in the box is the median
  • Whiskers extend to the minimum and maximum
  • If the median is closer to Q1 → positively skewed
  • If the median is closer to Q3 → negatively skewed
  • If the median is centred → approximately symmetric

✅ Advantages

  • Shows spread and skewness at a glance
  • Easy to compare multiple data sets
  • Shows outliers clearly

❌ Disadvantages

  • Doesn't show individual values
  • Doesn't show frequency
  • Can hide bimodal distributions

🎯 Measures of Central Tendency

Central tendency tells us the "typical" or "central" value in a data set.

Mean (Average)

Mean = Sum of all values / Number of values
x̄ = Σx / n

Properties:

  • Uses every data value
  • Affected by extreme values (outliers)
  • Best for data with no extreme outliers

Worked Example

Data: 5, 8, 12, 7, 3
Sum = 5 + 8 + 12 + 7 + 3 = 35
n = 5
Mean = 35 / 5 = 7

Median (Middle Value)

Arrange data in ascending order.
If n is odd: Median = middle value = ((n+1)/2)th value
If n is even: Median = average of (n/2)th and (n/2+1)th values

Properties:

  • Not affected by extreme values
  • Better than mean when data is skewed
  • Easy to find from stem-and-leaf or ordered data

Mode (Most Common Value)

Mode = value that appears most frequently

Properties:

  • Can have no mode, one mode, or multiple modes
  • The only measure for categorical data
  • Not affected by extreme values
  • May not be a useful measure if all values are different

Mean of Grouped Data

Mean = Σ(f × x) / Σf

where f = frequency of each class
x = mid-value of each class
1 Find the mid-value of each class interval: x = (lower + upper) / 2
2 Multiply each mid-value by its frequency: f × x
3 Sum all f × x values
4 Divide by total frequency Σf

When to Use Each Measure

MeasureBest Used WhenAvoid When
MeanData is symmetrical, no outliersData is heavily skewed or has outliers
MedianData is skewed or has outliersData is categorical
ModeData is categorical or you want the most common valueAll values are unique
💡 Exam Tip: When asked which measure is "most appropriate," consider if there are outliers (use median), if data is categorical (use mode), or if data is symmetric (use mean). Always explain why.

↔️ Measures of Spread

Measures of spread tell us how spread out the data is.

Range

Range = Maximum value − Minimum value
  • Simplest measure of spread
  • Affected by outliers
  • Only uses 2 data points

Interquartile Range (IQR)

IQR = Q3 − Q1
  • Measures the spread of the middle 50% of data
  • Not affected by outliers
  • Better measure of spread than range

Quartiles

  • Q1 (Lower Quartile): 25th percentile — 25% of data below this
  • Q2 (Median): 50th percentile
  • Q3 (Upper Quartile): 75th percentile — 75% of data below this

Percentiles

The Pk percentile means k% of the data falls below this value.

P10 = 10th percentile (10% below)
P90 = 90th percentile (90% below)
💡 Finding Quartiles from Data:
For n data points sorted in order:
Q1 = median of the lower half
Q3 = median of the upper half
(If n is odd, the median is excluded from both halves)

σ Standard Deviation

Standard deviation measures how spread out data is from the mean. A small SD means data is clustered near the mean; a large SD means data is spread out.

Formula for Ungrouped Data

SD = √[ Σ(x − x̄)² / n ]

Alternative formula:
SD = √[ (Σx²/n) − (Σx/n)² ]
SD = √[ (Σx²/n) − x̄² ]

Step-by-Step Method

1 Find the mean (x̄)
2 Subtract the mean from each value: (x − x̄)
3 Square each result: (x − x̄)²
4 Find the mean of the squared differences
5 Take the square root

Formula for Grouped Data

SD = √[ Σf(x − x̄)² / Σf ]

Alternative:
SD = √[ (Σfx²/Σf) − (Σfx/Σf)² ]

Where x = mid-value of each class, f = frequency

Interpreting Standard Deviation

  • Small SD: Data is closely clustered around the mean → consistent/uniform
  • Large SD: Data is spread out from the mean → varied/inconsistent
💡 Exam Context:
"A smaller standard deviation means the results are more consistent."
"A larger standard deviation means the results are more spread out/varied."

⚖️ Comparing Two Data Sets

Use the mean to compare central tendency and standard deviation to compare spread.

How to Compare in Exams

Always compare BOTH:
1. Central tendency (mean or median) — "On average, Set A is higher/lower than Set B"
2. Spread (SD or IQR) — "Set A is more consistent/varied than Set B"

Example Answer Framework

"The mean of Set A (70) is higher than the mean of Set B (65). This suggests that, on average, Set A performed better."
"The standard deviation of Set A (3.2) is smaller than that of Set B (15.8). This means Set A's results are more consistent, while Set B's results are more varied/spread out."

⚠️ Misinterpretation of Statistical Diagrams

Statistical diagrams can be misleading (intentionally or accidentally). Learn to spot common tricks!

Common Misleading Techniques

1 Broken / Truncated y-axis: Not starting the y-axis at 0 exaggerates differences between bars.
2 Unequal class widths: In histograms, unequal widths without frequency density makes some classes look more significant.
3 3D effects: 3D charts distort the visual perception of bar heights or slice sizes.
4 Pictograms with unequal symbol sizes: Larger symbols exaggerate the difference.
5 Inappropriate scale: A very large or small scale can minimise or exaggerate differences.
6 Misleading pie charts: Using angles/areas incorrectly or exploding slices.
7 Cherry-picking data: Showing only selected data points to support a narrative.
⚠️ Exam Tip: When asked "Why might this diagram be misleading?", look for: broken axes, missing labels, 3D effects, wrong chart type, or unequal scales.

🔍 Drawing Inferences from Statistical Diagrams

Drawing inferences means making conclusions or observations based on the data shown.

What to Look For

1 Trends: Is the data increasing, decreasing, or staying constant?
2 Clusters: Where is most of the data concentrated?
3 Outliers: Are there any unusual values far from the rest?
4 Comparisons: How do different groups compare?
5 Spread: Is the data tightly clustered or widely spread?
6 Symmetry / Skewness: Is the distribution symmetric or skewed?

Example Inference Statements

  • "Most students scored between 60 and 79 marks."
  • "There is a general increasing trend in sales from January to June."
  • "The data is positively skewed, indicating most values are on the lower end."
  • "Student A was more consistent as shown by the smaller IQR."
  • "The outlier at 15 suggests this could be an error or unusual result."
